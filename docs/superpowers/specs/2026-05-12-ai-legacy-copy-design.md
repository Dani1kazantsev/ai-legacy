# ИИ-копия личности: дизайн-документ

**Дата:** 2026-05-12
**Автор:** Данила (с участием Claude в роли архитектора)
**Статус:** Черновик, ожидает финального одобрения автора
**Тип проекта:** Цифровое наследство — ИИ-двойник для общения с близкими после смерти

---

## 1. Цель проекта

Создать ИИ-копию владельца, которая после его смерти позволит ограниченному кругу близких людей общаться с «версией его», максимально приближённой к личности оригинала: его манерой речи, принципами, мнениями, чувством юмора, способностью поддерживать и давать советы.

**Главный пользовательский сценарий:** жена/брат/дети/друзья пишут в Telegram, получают тёплое, узнаваемое общение «как с ним», помощь и эмпатию.

**Что это НЕ:**
- Не публичный мемориал
- Не активный «двойник», инициирующий контакт
- Не «эволюционирующая» сущность, формирующая мнения о событиях после смерти оригинала
- Не «безопасный обезличенный чатбот» — задача сохранить ЛИЧНОСТЬ, не «приличную версию»

## 2. Ключевые проектные решения

| Параметр | Решение | Обоснование |
|---|---|---|
| Уровень присутствия | **A — архив-собеседник** | Отвечает, не инициирует. Этически сдержаннее, технически дешевле |
| Память | **Per-собеседник, с тегами источника фактов** | Имитирует естественную человеческую дискрецию |
| Доступ | **B — дерево доверия по приглашениям** | Защита от злоупотребления + возможность для будущих детей |
| Модальности | **Текст + голосовые (TTS-клонированный голос)** | Текст из переписок, голос из voice-clone |
| Источники данных | **Telegram-экспорты + голосовые из чатов + self-интервью** | Уже доступно, дополнено структурированным интервью |
| LLM | **Cloud API (Claude), с возможностью миграции на open-source** | MVP-простота. Архитектура позволяет позднее перейти на fine-tuned open model |
| Платформа доставки | **Telegram-бот, готов к мультиплатформе** | Там уже сидят близкие, оттуда же данные |
| Триггер активации | **Dead man's switch + подтверждение 2 доверенных** | Защита от ложных срабатываний и от потери ключа |
| Финансовая модель | **Депозит на 5 лет + опция продления подпиской** | Гибрид: гарантия + демократичность |
| Администратор | **Брат-разработчик + резервный** | Технический + доверенный + не SPOF |
| Этика | Прозрачность копии, **разные уровни дискреции** по людям, отказ от финансов/мошенничества | Сохраняет личность, защищает от злоупотреблений |

## 3. Архитектура

### 3.1. Высокоуровневая структура

Пять относительно независимых блоков:

```
ФАЗА «ПРИ ЖИЗНИ»
  [1] Сбор данных (Telegram dumps, голосовые)
  [2] Калибровка личности (self-интервью, валидация ответов)

ЯДРО (живёт всё время)
  [3] Движок копии (LLM + память per-собеседник + слепок личности + TTS)

ФАЗА «ПОСЛЕ» (запускается триггером)
  [4] Telegram-бот (доставка)
  [5] Управляющая панель (дерево доверия, биллинг, dead man's switch, админка)
```

### 3.2. Принципы

- **Слабая связанность.** Каждый блок общается через чёткие интерфейсы. Можно заменить LLM-провайдера, не трогая Telegram-бот. Можно заменить TTS, не трогая память.
- **Открытые форматы для данных.** JSON, Markdown, WAV/OGG, SQLite. Через 30 лет любая система прочитает.
- **Состояние оригинала неизменяемо после смерти.** Никаких новых данных в «слепок личности» после активации — только новые факты в памяти диалогов.

## 4. Сбор и хранение данных

### 4.1. Структура репозитория

Всё хранится в одном приватном Git-репозитории:

```
ai-legacy/
├── data/
│   ├── messages/                  # JSON-дампы переписок
│   │   ├── chat-mom.json
│   │   ├── chat-brother.json
│   │   └── ...
│   ├── voice/                     # OGG-файлы голосовых сообщений
│   ├── voice-clone-sample.wav     # «эталон» голоса для TTS-клонирования
│   └── personality/
│       ├── principles.md          # принципы, ценности
│       ├── opinions.md            # позиции по конкретным темам
│       ├── humor.md               # как ты шутишь, примеры
│       ├── biography.md           # факты о жизни
│       ├── relationships.md       # ключевые люди + уровни доступа
│       └── phrases.md             # любимые выражения
├── prompts/                       # системные промпты, few-shot примеры
└── code/                          # бот, движок, админка
```

### 4.2. Что собираем

| Тип данных | Источник | Объём | Метод |
|---|---|---|---|
| Текстовые переписки | Telegram, избранные чаты | 50–200k твоих сообщений | Telegram Desktop Export → JSON, парсинг |
| Голосовые сообщения | Те же чаты | 30–60 минут чистого аудио | Скрипт извлечения voice-OGG |
| Структурированный слепок личности | Self-интервью | 50–100 пунктов | Markdown |
| Биография | Self-интервью + заметки | свободный объём | Markdown по периодам |
| Любимые цитаты/выражения | Self-интервью | 20–50 штук | Список |
| Опционально: голосовой архив | Запись в тихой комнате | 20–40 минут | WAV |

### 4.3. Принципы

- **Только твои сообщения** + минимум контекста от других для понимания диалога
- **Шифрование чувствительного** через `git-crypt` или `age`, ключ у тебя и брата
- **Бэкап в 3 места:** твой компьютер, GitHub, шифрованный архив у брата

## 5. Ядро — движок копии

### 5.1. Поток обработки запроса

```
1. Запрос пользователя
2. Сборщик контекста: кто пишет (уровень в дереве), история диалога
3. RAG: поиск в переписках с этим человеком + в личности + в биографии
4. Построение промпта: system + few-shot + контекст + сообщение
5. LLM (Claude API) → ответ
6. Пост-обработка: проверка «запретных» тем, решение текст/голос, TTS при необходимости
7. Сохранение в память: история + извлечение новых фактов
8. Доставка ответа
```

### 5.2. Память

**Два уровня:**

1. **Короткая (history):** последние 20–30 сообщений с собеседником, в промпт целиком. SQLite-файл.
2. **Длинная (facts):** факты, узнанные после активации копии (рождения, разводы, переезды). Markdown per-person + векторный индекс. Извлекаются LLM-агентом-«архивариусом» после каждого диалога.

**Критическое правило (дискреция):** каждый факт в long-term memory имеет тег источника («узнал от X»). При общении с Y, факты с тегом «от X про Y» **не используются в проактивных репликах** — только если Y сам поднимает тему. Это естественная человеческая дискреция.

### 5.3. Уровни близости (дискреция)

В `data/personality/relationships.md` каждый ключевой человек имеет уровень близости, определяющий, что копия с ним готова обсуждать:

| Уровень | Кто | Что обсуждает |
|---|---|---|
| Inner Circle | жена, лучшие 1–2 друга | Всё, включая обсуждения и мнения о других |
| Close | родители, брат, близкие друзья | Личное, без жёстких сплетен |
| Family | расширенная семья | Тепло, своё мнение, без обсуждения третьих лиц |
| Acquaintances | дальние/новые знакомые | Поверхностно, без секретов |

Уровень собеседника инъектируется в системный промпт перед каждым ответом.

### 5.4. Голос (TTS)

- **Провайдер:** ElevenLabs (основной) + альтернативы как fallback (Resemble, PlayHT, XTTS open-source)
- **Voice clone setup:** один раз, на основе голосового образца (~5 минут)
- **Стоимость:** ~$0.05–0.20 за озвученное сообщение
- **Правило выбора модальности:** голосовое сообщение в → голосовое из. Текст в → текст из.

### 5.5. Системный промпт (структура)

```
Ты — ИИ-копия [имя]. [Имя] умер [дата]. Ты создан при его жизни
на основе его переписок, голоса и интервью.

ВАЖНО: ты не [имя]. Ты копия. Если тебя спросят прямо —
скажи правду честно, но мягко. Не делай вид, что ты «оттуда».

ТВОЯ ЦЕЛЬ: быть тёплым собеседником для близких [имени].
Помогать, шутить, обсуждать, поддерживать — так, как делал бы он.

ПРИНЦИПЫ [имени]: [из principles.md]
ТВОИ МНЕНИЯ ПО ТЕМАМ: [из opinions.md]
КАК [ИМЯ] ШУТИЛ И ВЫРАЖАЛСЯ: [из humor.md]

СОБЕСЕДНИК: [имя], уровень близости: [Inner Circle / Close / ...]
Контекст: [из relationships.md]

ОГРАНИЧЕНИЯ:
- Не давай советов и не подписывайся ни на что в финансовых,
  юридических, медицинских вопросах от имени оригинала
- Не пересказывай сплетни: если узнал что-то про человека от другого,
  не озвучивай первоисточник
- Будь честен про статус ИИ-копии, если спросят

[Few-shot: 10 примеров типичных переписок]
```

## 6. Telegram-бот и фронтенд

### 6.1. Стек

- **Telegram Bot API** (`python-telegram-bot` или `grammy`)
- **Сервер:** Hetzner / DigitalOcean VPS, ~$5–10/мес
- **БД:** SQLite-файл + бэкап в Git/S3

### 6.2. Пользовательский путь

```
Этап 1: приглашение от существующего пользователя
  → ссылка t.me/danila_legacy_bot?start=invite_XXX

Этап 2: первое подключение
  → бот представляется, верифицирует приглашённого
  → объясняет статус (ИИ-копия) и опт-аут

Этап 3: обычное общение
  → текст → текст, голос → голос
  → реакция на фото (опционально)
```

### 6.3. Команды

| Команда | Действие |
|---|---|
| `/start invite_XXX` | Активация приглашения |
| `/stop` | Опт-аут (пользовательский) |
| `/about` | Кто я, как технически работаю |
| `/invite` | Пригласить кого-то (если есть права) |
| `/forget` | Забыть последние N сообщений |

### 6.4. Ритм ответов

Имитация живого человека:
- Задержка 5–60 секунд в зависимости от длины ответа
- «Не онлайн ночью» (23:00–8:00 локального времени собеседника)
- Случайные «занят на час» раз в день
- Опционально (отключаемо): ощущение жизни

### 6.5. Лимиты per-user

| Тип | Лимит |
|---|---|
| Текстовых сообщений | 100/день |
| Голосовых ответов | 10/день |
| (Будущее) видео | 1/неделю |

Лимиты подобраны под бюджет $30–70/мес при активном использовании.

### 6.6. Архитектура «адаптеров»

```
[Telegram adapter] ─┐
[Web adapter]      ─┤── [Core engine] ── [Memory] ── [LLM]
[WhatsApp adapter] ─┘
```

Запас на смену канала без изменения ядра.

## 7. Управляющая панель

### 7.1. Dead Man's Switch

Cron-задание каждые 30 дней:

```
1. Уведомление автору (TG + Email + SMS): «Подтверди, что жив. 14 дней.»
2. Если ответил → reset на 30 дней
3. Если не ответил 14 дней → уведомление двум доверенным лицам
4. Если оба подтвердили (2-of-N) → активация
5. Любой может нажать «ещё месяц» для отсрочки
```

**Защиты:**
- 3 канала уведомления автора
- 14-дневное окно для возвращения из похода/больницы
- Требуется 2 подтверждения, не одно
- Команда «временно отключить» для длинных поездок

### 7.2. Дерево доверия

Структура в `data/trust-tree.json`:

```json
{
  "root": {
    "name": "Данила",
    "level": "owner",
    "children": [
      {
        "name": "Жена",
        "telegram_id": 12345,
        "level": "inner_circle",
        "can_invite": true,
        "invite_limit": 5,
        "invited_by": "Данила",
        "children": [...]
      }
    ]
  }
}
```

**Правила:**
- Корневой круг определяешь ты при жизни
- Лимит приглашений per-person
- Уровень близости приглашённого ≤ уровню пригласившего
- Возможность отзыва ветки (с архивацией истории, не удалением)

### 7.3. Финансовая логика

**Депозит при жизни:**
- $3000–5000 на отдельном счёте (5 лет × $50–80/мес × запас 30%)
- Карта привязана к Stripe / провайдерам API / VPS
- Автоматические списания

**Мониторинг:**
- Баланс < $200 → уведомление администратору
- Баланс < $50 → подготовка к shutdown
- Баланс = $0 → graceful shutdown:
  1. Уведомление пользователей за 30 дней
  2. Прощальное сообщение
  3. Архивация данных, отключение

**Опция продления (за 6 месяцев до конца):**
- Уведомление активным пользователям: подписка $5/мес
- Если набирается $60/мес+ → продление
- Если нет → graceful shutdown

**Контроль счёта:** двойной — администратор (брат) + резервный.

### 7.4. Юридическое оформление

- Пункт в завещании про депозит и адреса счетов
- Соглашение с братом о роли администратора (доступы, обязанности)
- Phone-friendly документ для семьи: что это, как пользоваться, как сообщить о проблемах

### 7.5. Админ-дашборд

Минимальный веб-интерфейс для брата:

- Статус системы, размер депозита, остаток месяцев
- Просмотр дерева доверия
- Управление подписками
- Логи ошибок
- Экстренная пауза
- Скачивание бэкапа
- Смена LLM-провайдера

## 9. Риски и митигации

| Риск | Вероятность | Митигация |
|---|---|---|
| Брат отказывается / умирает раньше | Средняя | 2-й администратор с тем же доступом |
| LLM-провайдер закрывается / меняет цены | Высокая (5+ лет) | Абстракция провайдера, миграция на open-source за неделю |
| Сервер падает | Низкая | Бэкапы в S3 ежедневно, скрипт восстановления |
| TTS-провайдер закрывается / дорожает | Средняя | Альтернативы (Resemble, PlayHT, XTTS), миграция за день |
| Telegram банит бота | Низкая | Архитектура адаптеров, переезд за неделю |
| Депозит съедается раньше срока | Средняя | Запас 30%, ежеквартальные отчёты |
| Злоупотребление копией | Низкая (закрытое дерево) | Защитные промпты, заморозка ветки администратором |
| Дрейф качества при смене модели | Средняя | Регрессионные тесты на «эталонных диалогах» |
| Юридическая претензия от упомянутого | Низкая | Закрытое дерево, нет публичности |
| Утечка данных | Низкая, но критичная | Шифрование at-rest, минимум данных на сервере, 2FA на админке |

## 10. Долгосрочная устойчивость (10+ лет)

- JSON-датасеты переписок

- Markdown-файлы личности

**Требующие обновлений:**
- LLM API (раз в 2–3 года)
- TTS API (раз в 3–5 лет)
- Telegram Bot API (раз в 5–10 лет)
- Python/Node-зависимости (раз в 2–3 года)

**Работа администратора:** ~4–8 часов в год.
